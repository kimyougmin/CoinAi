환경 선정 이유 및 프로젝트 관리

### React, TypeScript, Node.js
* React
  * 채팅 기능을 구현할 때 리랜더링 관리를 위한 컴포넌트화
  * 추후의 유지보수 및 재사용성
  * 불필요한 랜더링을 차단하고 필요한 컴포넌트만 리랜더링을 함으로 일관성 유지

* TypeScript
  * js의 난해한 오류 방지
  * 타입 파일을 별도로 정의하여 프로젝트 구조 간략화
